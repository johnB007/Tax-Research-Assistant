# Security

If you discover a security vulnerability, please email security concerns to the maintainer rather than using the issue tracker.

Please do not publicly disclose the vulnerability until a fix is available.

## Responsible disclosure

1. Email details of the vulnerability
2. Allow time for a fix to be developed and released
3. Coordinate with maintainers on disclosure timing

## Scope

This project is for research and education. Always verify security best practices when using in production environments.
